AI-Powered RAG Chatbot

An intelligent, multilingual chatbot providing instant answers from a university knowledge base using Retrieval-Augmented Generation.

View Live Demo

Key Features

This project combines several modern AI techniques to deliver a robust user experience.

Multilingual Support

Handles user queries and provides responses in both English and Bengali, automatically detecting the input language.

Retrieval-Augmented Generation

Combines efficient knowledge retrieval with a powerful language model to generate accurate, context-aware answers.

Efficient Knowledge Retrieval

Utilizes FAISS (Facebook AI Similarity Search) for millisecond-fast similarity search across text embeddings.

Scalable Architecture

Built with a lightweight Flask backend, ready for deployment as a scalable web service on platforms like Hugging Face.

Lazy Loading

Heavy AI models are loaded only on the first user request, ensuring fast application startup times.

Meaningful Embeddings

Employs Sentence-Transformers to convert text into rich numerical representations that capture semantic meaning.

How It Works: The RAG Workflow

From a user's question to a generated answer, the system follows a precise, multi-step process.

1

User Query & Language Detection

The user submits a question. The system first detects if the query is in English or Bengali to select the correct knowledge base.

2

Query Embedding

The user's query is converted into a numerical vector (embedding) using the SentenceTransformer model, capturing its semantic meaning.

3

Context Retrieval (The "R")

The query embedding is used to search the FAISS vector index. The most semantically similar text chunks from the knowledge base are retrieved as context.

4

Prompt Construction

A detailed prompt is constructed, combining the original query with the retrieved context and instructing the LLM to answer based only on this information.

5

Answer Generation (The "G")

The complete prompt is sent to Cohere's `command-r-plus` model, which generates a coherent and factually grounded answer.

6

Display to User

The final generated answer is sent back through the Flask API and displayed to the user in the web interface.

Knowledge Base Composition

The chatbot's knowledge is sourced from two distinct text files, one for each supported language. This visual represents the bilingual foundation of the system, ensuring it can cater to both English and Bengali-speaking users by retrieving information from the appropriate source.

The project requires Python and pip for dependency installation (`Flask`, `cohere`, `faiss-cpu`, etc.). A `.env` file is needed to store the `COHERE_API_KEY`. The application is then launched by running `python app.py`.
Future plans include adding chat history for follow-up questions, providing source citations for answers, and implementing a user feedback loop to continuously improve the model's performance and knowledge base.
